

①github上で、MLアカウントのこれからコミットしたい対象のリポートリポジトリを
　フォークして、自分のアカウントのリモートリポジトリとして複製する

②自分のアカウントのリモートリポジトリから
　クローンして、ローカルのローカルリポジトリとする。(git clone)

③編集用のブランチを作成して(git branch)
　そちらのブランチに切り替える(git checkout)

④コミットが必要な成果物一式をフォルダに配置する

⑤成果物一式をステージングエリアに登録する(git add)
　ステージングエリアに登録できたことを確認する(git status または git diff --cached)

⑥ステージングエリアの成果物一式をコミットする(git commit)
　コミットできたことを確認する(git status)
　
⑦ローカルリポジトリの内容を、自分のアカウントのリモートリポジトリにプッシュする。
　(git push origin "編集用のブランチ名")

⑧git hubのプルリクエストメニューから
　新しいプルリクエストを選択する。
　プルリクエストを以下の通り設定した後、確定する。
　・【ベースブランチ】 ← 【トピックブランチ】を選べるようになっているので、
　　【ベースブランチ】：ＭＬアカウントの対象リモートリポジトリのmasterブランチ
　　【トピックブランチ】：自分のアカウントの対象リモートリポジトリの "編集用のブランチ"
　　を選択する。
　・コメントを書く
　・Reviewersにレビュアー(ヒマンシュさん)を設定する

　※トピックブランチ：レビューしてもらいたい対象
　※ベースブランチ：マージしたい先

⑨(ヒマンシュさん作業)
　プルリクエストを受けて、変更内容を確認する
　OKであれば、承認する。
　
⑩承認がえられたら、
　Merge pull requestボタンで、
　編集用ブランチをMasterブランチに合流させる。
